# VoxFree 0.5.0 — Voice Selector in Indicator

## Goal
Add an in-indicator voice selector so users can switch TTS voices without leaving the desktop. Voice changes stop current playback and persist system-wide.

## Architecture

Core (bash) ←writes→ /tmp/voxfree/state ←reads→ UI (indicator)
   readloud.sh              STATE=playing          Python/GNOME extension
   set-voice.sh             PID=1234               (subprocess calls only)
   list-voices.sh           LAST_TEXT=...

Strict separation:
- Core scripts never import GTK/GNOME/JS — pure bash
- UI scripts never touch config files or state files — only call core scripts via subprocess

## New files

### 1. lib/list-voices.sh

Shared bash helper that lists all Mimic 3 voices with their status.

**Output format** (one line per voice, parseable):
```
1|en_UK/apope_low|1
2|en_UK/semaine|0
3|en_US/cmu-arctic_low|1
```
Fields: `INDEX|voice_name|is_installed (1/0)`
Current voice is marked by comparing against `~/.config/voxfree/voice` / `/etc/voxfree/voice`.

**Logic:**
- Run `mimic3 --voices 2>/dev/null` and filter `en_UK` / `en_US` lines
- Check installed status via voice directory existence:
  - `/usr/share/mycroft/mimic3/voices/$LANG/$NAME`
  - `$HOME/.local/share/mycroft/mimic3/voices/$LANG/$NAME`
- Compare each voice against current config to mark the active one
- Fallback: if `mimic3` not found or no voices listed, default to `en_UK/apope_low`

### 2. voxfree-set-voice.sh

Thin wrapper that writes the selected voice to user config.

**Logic:**
```bash
VOICE="$1"
USER_CONF="$HOME/.config/voxfree/voice"
mkdir -p "$(dirname "$USER_CONF")"
echo "$VOICE" > "$USER_CONF"
```

No UI output required — called silently from indicator.

## Modified files

### 3. ReadLoud/voxfree-indicator (Python)

**Additions:**
- `_voiceItems = []` — store menu item references for rebuild
- `_build_voice_menu()` — runs `list-voices.sh`, parses output, creates `Gtk.MenuItem` for each voice
- Insert `Voices` submenu between "Replay Last" and the separator
- Voice menu items show: `✓  voice_name` (checkmark on current voice, plain on others)
- On voice selection handler:
  1. Run `voxfree-stop-all` (stops current playback)
  2. Run `voxfree-set-voice $VOICE` (saves choice)
  3. Rebuild menu (updates checkmark position)

**Menu structure:**
```
Read Aloud
Stop Reading
Replay Last
Voices
  ✓ en_UK/apope_low
    en_UK/semaine
    en_US/cmu-arctic_low
    ...
─── separator ───
Quit
```

### 4. ReadLoud/gnome-shell-extension/extension.js

**Same pattern:**
- `_voiceItems = []` — store menu references
- `_buildVoiceMenu()` — spawn `list-voices.sh`, parse output, build `PopupMenuItem` entries
- Insert as submenu item (use `PopupSubMenuMenuItem` or separator + items)
- On activation: spawn `voxfree-stop-all`, then `voxfree-set-voice $VOICE`, then `_updateUI()`

### 5. ReadLoud/readloud.sh (installer)

- Copy `lib/list-voices.sh` to `$STATE_LIB_DEST/` (alongside `state.sh`)

## Behavior details

| Action | Effect |
|--------|--------|
| Open indicator → click Voices | Flat list of all Mimic 3 voices |
| Current voice shown | Preceded by `✓` checkmark symbol |
| Select a voice | Stops current TTS playback → saves config → rebuilds menu |
| New voice takes effect | On next "Read Aloud" or keyboard shortcut |
| Voice persists | Written to `~/.config/voxfree/voice`, survives restart |

### Edge cases
- **mimic3 not installed:** Show "Install Mimic 3 first" message
- **Voice not installed:** Indicator does NOT auto-download (unlike CLI `--voice`). Show note: "Download via: voxfree --voice"
- **Indicator not running:** Voice change still works via `voxfree --voice` CLI
- **Many voices (30+):** GTK/GNOME menu scrolls natively — no pagination needed
